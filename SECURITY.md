# Security Policy

## Supported Versions

Because `secure-passkeys` handles authentication and sensitive security data, we strongly recommend always updating to the latest stable release. We only provide security patches for the most recent major versions.

| Version | Supported          | Notes                                     |
| ------- | ------------------ | ----------------------------------------- |
| 1.0.x   | :white_check_mark: | Current Stable Release (Fully Supported)  |
| < 1.0   | :x:                | Outdated - Please upgrade immediately     |

> **Note:** Please ensure your WordPress core installation and PHP environment are also kept up to date to ensure passkey encryption functions securely.

## Reporting a Vulnerability

We take the security of this plugin very seriously. If you find a security vulnerability, **please do not open a public GitHub issue or discussion.** Public reports put all sites using this plugin at risk.

### How to Report

1. **GitHub Private Vulnerability Reporting (Recommended):** Click on the **Security** tab at the top of this repository page, click **Vulnerabilities** on the left sidebar, and click **Report a vulnerability**. This securely sends the report straight to the maintainers.
2. **Email:** Alternatively, you can reach us privately at **mohamed.endisha@gmail.com**.

### What to Expect From Us
* **Response:** We will acknowledge receipt of your report within **48 hours**.
* **Fix Timeline:** Validated vulnerabilities will be patched within **7–14 days**.
* **Coordinated Release:** We will release the fix simultaneously on GitHub and the WordPress.org Plugin Directory, giving credit to the researcher (if desired) once the patch is live.

Thank you for helping keep the WordPress passkey ecosystem safe!
